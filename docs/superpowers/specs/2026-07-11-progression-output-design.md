# 养成材料与产出设计（第 2 部分：喂养管道与经济流速）

日期：2026-07-11
状态：待用户审阅
范围：养成数值框架四步定案的第③步——每条养成线的"养料从哪来、多快到手"。承接第 1 部分 `2026-07-11-progression-framework-design.md`（属性系统+组成+模块成长，已实施暂存）。**本 spec 只定管道结构和体验锚点，全部具体数值（掉率/成本/经验值）由子计划 B 的生成式框架按锚点反解**，不手填。

## 1. 总原则

- 每个养成模块有且只有明确的喂养管道；流速 = 毕业预期 ÷ 第一章总局数（中度锚：台阶关 4/7/10 各回刷 5~10 局，全程 25~40 局）。
- 在线与离线同源：离线收益继续走真实规则模拟（`OfflineCombatService` 按 chest 规则产宝箱），不另立一套数。
- 毕业快照（反解的终点状态）：稀有~史诗 Lv≈12 整套 + 8~10 颗均值 2 级宝石 + 4~5 条生效铭文 + 角色 Lv≈12。

## 2. 经验管道（角色等级线，20% 战力份额）

- **来源**：杀怪经验（`EnemyTypes.exp`），全体上阵角色各一份全额（现状保留）；离线同源。
- **曲线（100 级分段几何）**：
  - 1~30 级：×1.15/级（现状形状），`expBase` 由框架反解，目标 = **顺推+正常中度回刷自然到 Lv12 毕业，不需要专门刷级**；
  - 31~60 级：增长率降档（占位 ×1.08/级）；
  - 61~100 级：再降档（占位 ×1.04/级）——长线追求，防纯几何到 100 级需要天文数字经验。
  - 段点与比率是 balance.xlsx 的锚点旋钮；`expToNext` 实现改为分段查表计算。
- 现状核对：第一章毕业约需累计 1200 经验（×1.15 曲线），单局期望 30~60 exp、25~40 局约 1000~1900——现曲线形状基本落在锚点内，反解只需微调 `expBase`。

## 3. 装备管道（50% 份额，三管道分工）

| 管道 | 角色 | 期望件数占比 |
|---|---|---|
| 关卡胜利掉落（`drop.xlsx`） | 保底：打就有 | ~50% |
| 宝箱装备（`ChestService`） | 惊喜：随机性峰值 | ~30% |
| 合成（`craft.xlsx`） | 定向兜底：非酋保险丝 | ~20% |

- 合成定价锚点：**一次合成的打造石成本 ≈ 台阶期 5~8 局的期望打造石收入**（N 反解）；tier_1 精调，tier_2/3 曲线外推标 provisional。
- 合成品质权重按"合成期望品质 ≈ 同期掉落中位品质偏上半档"反解（合成是兜底不是抽奖机）。

## 4. 宝石管道（17% 份额，宝箱专属）

- 普通箱：1 颗 1~2 级；Boss 箱：1 颗 2~3 级；章节箱：2 颗 3~4 级（结构沿用，档位重定）。
- **5~6 级宝石第一章不产出**——长线追求位，第二章再开管道。
- 掉率反解目标：毕业插满 8~10 颗、均值 2 级。

## 5. 铭文卷轴管道（8% 份额，稀缺品）

- Boss 箱 / 章节箱专属（普通箱不出）；洗练要肉疼，随机 roll 的乐趣才成立。
- 反解目标：毕业期望摸到 7~9 张卷轴（4~5 条生效 + 若干次洗歪）。

## 6. 打造石管道

- 来源：宝箱固定携带（现状）+ **出售装备返还**（回收率锚点 = 合成价的 15~20%，让垃圾装备有价值）。
- 现状核对：出售目前只给金币；"出售返打造石"是新机制，实施时进 `InventoryModel.sellItem/sellBatch` 的返还包。

## 7. 宝箱本身（结构不变，数值反解）

- 三层结构沿用：小怪掉率 `mobChance` / 关底 `finalChance` / 箱型权重 `typeWeights`（chest.xlsx）。
- **机制补课（本 spec 唯一代码实施项）**：`ChestService.CHEST_REWARD_PROFILE` 硬编码 → 表化进 `chest.xlsx` 新 sheet `Rewards`：
  - 列：`chestType, equipmentRolls, forgeStoneMin, forgeStoneMax, gemCount, gemLevelMin, gemLevelMax, scrollMin, scrollMax`；
  - `ChestService` 改读配置；现值原样搬入（行为等价，同 seed 同结果）；离线链路自动复用。
  - 不表化，框架的 derive 就够不着宝石/卷轴/打造石三条管道的源头。

## 8. 金币

- 现状金币主要来自出售与结算，消费端未成型（无商店）。本轮**不设计金币经济**，只在框架份额表留列，第二章随商店/技能升级一起定。

## 9. 不做 / 后置

- 难度导出（怪物强度曲线）与推进模拟验证 → 第④步节奏 spec。
- 全部具体数值 → 子计划 B `balance:derive` 反解写回。
- 宝石升级合成、金币经济、第二章管道 → 第二章规划。

## 10. 验收标准

- `Rewards` 表化后 `npm run verify` 全绿；`ChestService` 同 seed 开箱结果与表化前逐位一致（现值搬运）。
- 经验曲线分段实现后：`expToNext` 单测覆盖段点连续性（段边界不跳变、单调递增）；Lv100 累计经验为有限合理值。
- 出售返打造石：返还进 `RewardBundle.materials`，锁定/已穿不可卖规则不变，单测覆盖。
