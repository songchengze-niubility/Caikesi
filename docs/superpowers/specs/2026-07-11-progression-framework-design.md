# 养成数值框架设计（第 1 部分：属性系统 + 养成组成 + 模块成长）

日期：2026-07-11
状态：待用户审阅
范围：数值平衡大任务的子项目①前半。**本 spec 只定"玩家侧养成本体"**——属性维度、双层算法、模块占比、各模块成长结构，以及承载它们的生成式框架形态。经验曲线数值、掉落/材料产出流速、怪物难度导出、节奏模拟验证全部**后置到后续 spec**（用户指示：先定养成，再定养成需要的材料，最后算节奏）。

## 1. 背景与问题

全游戏数值散在 battle/equip/drop/chest/inlay/craft 六张 xlsx，上百格占位值互相咬合（例：1 级攻宝石 +30 ≈ 半把武器）。直接逐格调参会越堆越多、账算不清。用户决策：自底向上重建——先定养成组成，再定模块成长，再定产出，最后节奏是算出来的结果。

## 2. 已拍板的决策（对话记录，2026-07-11）

| 决策 | 内容 |
|---|---|
| 刷关强度锚点 | 中度：台阶关 4/7/10 各回刷 5~10 局，第一章全程 25~40 局（后置的节奏 spec 用） |
| 真源模式 | **生成式**：曲线/预算成为新真源（balance.xlsx + 纯 TS 模块），派生数值由导出脚本写回各 xlsx；手感例外走 Overrides 表，不直改派生格 |
| 定案顺序 | ①养成组成 → ②模块成长 → ③掉落产出 → ④算节奏。本 spec 覆盖①② |

## 3. 属性系统（18 维 + 1 功能位）

### 3.1 双层算法

```
一级属性面板 = (白板 + 固定值加成合计) × (1 + 百分比加成合计)
```

- 固定值层负责"数字变大"（装备主词条、宝石）；百分比层负责"倍率放大"，百分比作用于**全池**（白板+固定）。三围百分比（生命%/攻击%/防御%）**归属二级属性池**：角色等级模块按等级给三围同幅百分比，装备词条/铭文从二级池随机 roll 到单项百分比。
- 现状差距（实施时改）：`charLevelCoef` 目前只放大白板 hp/atk；铭文没有 atk%/hp% 词条；`EffectiveStats` 叠算顺序需按本公式重排。

### 3.2 一级属性（3 个，固定值形态）

| 属性 | 固定值来源 |
|---|---|
| 生命 hp / 攻击 atk / 防御 def | 白板、装备主词条、宝石 |

一级属性的百分比形态（生命%/攻击%/防御%）见 3.3 二级属性池。

### 3.3 二级属性（17 个，进乘区；其中三围百分比是叠算层形态，不新增面板字段）

| 属性 | 定义/战力折算 | 来源 | 上限（占位，验证后修正） |
|---|---|---|---|
| **生命% / 攻击% / 防御%（归入二级池）** | 各自放大对应一级属性全池 | 角色等级（模块化三围同幅）、装备词条、铭文 | 词条+铭文合计各 40%（角色等级模块不计入上限） |
| 暴击率 critRate | DPS ×(1+暴率×暴伤) | 装备词条、宝石、铭文 | 50% |
| 暴击伤害 critDmg | 与暴率联动 | 铭文为主 | 150% |
| 攻速 attackSpeed | DPS 直接倍率 | 史诗+装备词条、铭文（稀有维度） | +50% |
| 移速 moveSpeed | 固定+百分比双形态：`(白板+固定)×(1+%)`；行军/冲锋/走位 | 装备词条、铭文（小额） | +40% |
| **技能急速 skillHaste（新增）** | 技能触发进度恢复速度：计时型计时 ×(1+急速)；计数型所需普攻次数 ÷(1+急速)（向上取整，保底 1） | 铭文为主、史诗+装备词条 | +50% |
| 闪避 dodgeRate | EHP ÷(1-闪避) | 装备词条 | 30% |
| 格挡率 blockRate | EHP（×格挡减伤联动） | 装备词条（坦克向） | 40% |
| 格挡减伤 blockRatio | 同上 | 白板/职业特性为主 | 70% |
| 伤害减免 dmgReduce | EHP ÷(1-减免) | 铭文 | 40% |
| **普攻伤害加成（新增）** | 只吃普攻（含普攻弹道） | 装备词条、宝石 | 40% |
| **技能伤害加成（新增）** | 只吃技能效果（技能弹道/场地/DoT 均属技能） | 装备词条、宝石 | 40% |
| **单体伤害加成（新增）** | 本次结算目标数=1 时生效 | 铭文 | 40% |
| **群体伤害加成（新增）** | 本次结算为 aoe/多目标时生效 | 铭文 | 40% |
| 全伤害加成 dmgBonus | 无条件生效（现有 dmgBonus 语义收窄为"全伤害"） | 宝石、铭文 | 60% |

**伤害类型标签**：每次伤害结算带两个标签——来源（普攻/技能）×范围（单体/群体），攻击管线自动打标。**同乘区加算**：最终加成 = 全伤害 +（普攻或技能）+（单体或群体），整体 ×(1+合计)，三类不互乘。

### 3.4 功能位（不进养成池）

攻击距离 range：管站位与职业身份，只留白板 + 技能/Buff，不进任何随机词条池。

## 4. 养成组成（第一章毕业快照）

占比 = 能过 L10 的达标玩家总战力（战力口径 = EHP×DPS，18 维全折算）中各模块份额：

| 模块 | 占比 | 角色 |
|---|---|---|
| 白板（基础属性） | **5%** | 只是"能站上场"；职业差异化载体 |
| 角色等级 | 20% | 自然增长线（百分比放大全池） |
| 装备 | 50% | 主动经营大头，台阶卡点主轴（固定值） |
| 宝石 | 17% | 副轴大头（固定值专精） |
| 铭文 | 8% | 副轴小头，随机 roll 追求（百分比） |
| 技能升级 | 0%（留列） | 第二章再分配 |

毕业玩家 ≈ 白板 20 倍强；怪物曲线（后置 spec）需同步拉开约 20 倍。绝对数值锚点：**沿用现有职业白板作为基准**，其余全部由占比反解。

## 5. 模块成长结构（形状定死，绝对值由框架反解）

### 5.1 角色等级
- 上限 **100 级**；第一章毕业预期 ~Lv12。
- 每级给 hp%/atk%/def% 三围同幅线性百分比。
- 经验曲线**本 spec 不定**（用户指示后置）；仅记录约束：现行纯几何 ×1.15 撑不到 100 级，届时改分段（前期几何、中后期线性）。

### 5.2 装备
- 主词条 = 部位基数 × 品质倍率 × 等级系数：品质五档（白→优→稀有→史诗→传说）等比阶梯（相邻约 ×1.35，反解）；等级线性系数；部位分工沿用现状（武器攻，头/胸/腿血防）。
- 随机词条：稀有+1 / 史诗+2 / 传说+3 条，从二级属性池抽（三围百分比、普攻/技能伤、暴击、攻速、移速等）。
- **穿戴等级需求（新增机制）：需求 = 装备等级**，仅穿戴时校验；老存档已穿戴的不强制脱，读档自愈只补字段。
- 毕业预期：稀有~史诗、Lv≈12 整套。

### 5.3 宝石
- **1~6 级**，每级价值等比（约 ×1.6，反解）；类型专精：攻/血/防→一级属性，暴击/伤害类→二级属性。
- 获取即用，第一章**不做**宝石升级合成；第一章产出集中在低级段，5~6 级是长线追求。
- 毕业预期：8~10 颗、均值 2 级。
- 实现红线：`MaterialId` 有限联合 `gem_<type>_<1|2|3|4>` 与 `Gems.maxLevel` 耦合，上限 4→6 要同步拓宽联合类型（代码地图已标注）。

### 5.4 铭文
- 每条 = 随机属性 + 区间随机值，**区间下限 = 上限的 60%**；洗练走现有卷轴覆盖重抽，不加保底。
- 毕业预期：4~5 条生效。

### 5.5 白板
- 不成长；各职业 18 维白板按定位分布（坦克血防格挡、输出攻暴、医师均衡），总量卡 5% 份额。

## 6. 框架实现形态（生成式）

```
balance.xlsx（真源：占比/曲线形状/上限等锚点旋钮）
  → tools/balance-model/（纯 TS：power.ts 战力口径 / budget.ts 份额反解 / derive.ts 写回+Overrides）
  → npm run balance:derive 写回 battle/equip/inlay/craft 等 xlsx 的派生列
  → npm run config（现有导表不动）→ 现有验证链
```

- `balance:check`：derive 到临时文件与现 xlsx 比对，漂移即红，挂进 `npm run verify`（与 `.generated.ts` 防漂移同模式）。
- Overrides 表：手感例外（表/键/字段/值/理由），派生最后一步应用。
- 校验：占比合计 100%、Override 引用存在、derive 幂等（两次零 diff）、二级属性钳制上限。
- 本部分 derive 范围：**玩家侧表**（equip/inlay 数值、白板、等级系数）。怪物/掉落/合成的派生等产出与节奏 spec 定稿后再接入。

## 7. 涉及的代码改动（实施计划展开）

1. `CombatStats` 13→18 维（技能急速 + 4 个伤害类型加成；dmgBonus 语义收窄）；Excel Stats/EnemyTypes 列同步。
2. 伤害管线打标：`calcDamage`/`Effects.applyEffect` 增加来源×范围标签；普攻/技能弹道/场地/DoT 归类。
3. `SkillRuntime` 接技能急速（计时/计数两型）。
4. `EffectiveStats` 重排为双层公式；装备词条/铭文池增三围百分比词条；等级系数改乘全池。
5. `EquipItem` 穿戴等级需求 + 穿戴校验 + UI 灰化 + 读档自愈。
6. `charGrowth.maxLevel` 30→100；`Gems.maxLevel` 4→6（含 MaterialId 联合类型拓宽）。
7. `tools/balance-model/` + balance.xlsx + `balance:derive`/`balance:check` npm 脚本。
8. Buff/词条/宝石各属性联合类型拓宽 + 全量单测补铺。

## 9. 验收标准

- 18 维属性在类型/导表/校验/单测全链贯通，`npm run verify` 全绿。
- `balance:derive` 幂等且产出通过占比校验：按毕业快照（稀有~史诗 Lv12 套 + 8~10 宝石 + 4~5 铭文 + Lv12）折算战力，各模块份额与第 4 节表偏差 ≤2 个百分点。
- 伤害标签单测：普攻/技能/单体/群体四类加成各自只在对应场景生效。
- 现有 13 项 pacing 门槛在数值重导**之前**保持全绿（属性系统扩展本身不改现有数值语义）。
