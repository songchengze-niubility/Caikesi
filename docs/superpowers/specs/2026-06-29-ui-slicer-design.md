# ui-slicer 设计文档

> 日期:2026-06-29 · 状态:已实现
> 一句话:一个自包含的本地 HTML 页,手动框选 UI 组件,可做轻量去底/手工遮罩,导出 PNG 拿去 Figma 继续做。

## 背景 / 动机

用户想要一个本地"抠 UI"小工具:把一张 UI 图丢上画布,框选组件,裁出来,再导入 Figma 加工。灵感来自 studio.vberai.com 的 UI 拆分功能。

**重要历史教训(本设计的核心约束来源)**:本项目此前已经建过一个极其庞大的同类工具(`asset_workbench/tools/manual_ui_extractor/` + `ai_segment_service`),从没进 git、后按用户要求删除、无法恢复。它失控的两个根因:

1. **过度工程**:从"框选裁切"滚成了 AI 全图拆分、VLM 探测、alpha 质检、审查包 ZIP、候选树、组件库 Prefab 草稿、Cocos 挂载片段生成……设计日志里近百行决策。
2. **核心难题没解决**:扁平截图做矩形裁切去不掉底;真正想救场的 AI 抠图(VLM)从头到尾没接通。

用户已明确:那些记忆是"坑",本次只要一个小而可控的本地手动工具,产出 PNG 导入 Figma。本设计据此把范围收住:不再做 AI 自动拆解和 Cocos 衔接,只在浏览器 Canvas 内做人工框选、轻量去底和遮罩修边。

## 目标

- 浏览器打开一个本地 HTML 页即可用,**无服务、无构建、无第三方依赖**。
- 拖入一张 UI 图,手动框选若干矩形组件。
- 每个框裁成 PNG；默认可做边缘 flood-fill 去底,也可关闭后保留矩形背景。
- 支持吸管取底色、每框容差、手工遮罩画笔,用于修掉自动去底不干净的区域。
- 通过**下载文件**、**复制到剪贴板**或可选写入本地文件夹交付,方便拖入或粘进 Figma。
- 框的增删改、移动、缩放、像素级微调、命名。

## 非目标(明确不做,防重蹈覆辙)

AI / VLM / SAM / rembg、语义级自动拆图、alpha 质检、审查包、ZIP 打包、layout.json、工程文件存盘、组件库 / Prefab、Cocos 衔接生成、任何本地服务进程。这些是上一版失控的根源,本版一律不碰。当前仅允许 Canvas 内的轻量边缘去底和人工遮罩修边。

## 用户工作流

1. 浏览器双击打开 `tools/ui-slicer/index.html`。
2. 拖一张 UI 图进画布(或点"打开图片")。
3. 在图上拖拽画框,框住要的组件;按需移动/缩放/微调/命名/删除。
4. 每个框按需要调整"去底/吸管/容差/画笔遮罩",右侧预览就是导出结果。
5. 对某个框点"下载 PNG"得到文件,或点"复制"直接到剪贴板。也可"下载全部"。
6. 在 Figma 里拖入文件,或 Ctrl+V 粘贴,继续加工。

## 界面与交互

布局:
```
┌──────────────────────────────────────────────────┐
│ [打开图片]   缩放 100% [适配]        [下载全部]    │ 顶栏
├───────────────────────────────────────┬──────────┤
│            画布(图 + 框)               │ 组件列表  │
│  空白处拖 = 画新框                       │ ① comp_1 │
│  框内拖 = 移动 / 拖角手柄 = 缩放          │ ② comp_2 │
│  滚轮缩放(对准光标)/ 空格+拖 = 平移      │  ...     │
│  方向键 = 微调 1px(Shift = 10px)        │──────────│
│  Del = 删除选中框                        │ 选中框:  │
│                                         │ 名 [___] │
│                                         │ x y w h  │
│                                         │ [下载][复制][删除] │
└───────────────────────────────────────┴──────────┘
```

交互细节:
- **画框**:在空白画布区按下并拖动 → 新框,自动命名 `comp_N`,创建即选中。
- **选中**:点框体或点列表项;选中框显示 8 个缩放手柄。
- **移动**:框体内拖动。**缩放**:拖手柄。**微调**:方向键 1px,Shift+方向键 10px。
- **命名**:列表/选中面板里编辑名字,作为下载文件名(`<名>.png`)。
- **删除**:Del 键或"删除"按钮。
- **缩放/平移**:滚轮对准光标缩放;空格+拖 或 中键拖 平移;"适配"一键把整图塞进视口。
- 所有框坐标在交互结束时对齐到**源图整数像素**,保证裁切精确。

## 技术结构(单文件,内部清晰分区)

一个 `tools/ui-slicer/index.html`,CSS/JS 内联。虽是单文件,JS 按职责分成独立函数块,便于以后维护或拆分:

1. **图片加载**:拖放 / `<input type=file>` → 解码为 `ImageBitmap`/`Image`,记录原始宽高;无图时禁用导出类控件。
2. **viewport(坐标变换)**:维护 `scale` 与 `offset`,提供 `screenToImage()` / `imageToScreen()`;缩放平移只改这两个量,不动框数据。
3. **box store(框模型)**:`boxes: [{ id, name, x, y, w, h }]`,源图像素空间;增/删/改/选中 API;每次变更后写 localStorage(按图片名做 key),刷新自动恢复——纯保护,无 UI 暴露。
4. **渲染**:`requestAnimationFrame` 重画:底图 → 所有框矩形 → 选中框手柄。纯函数式,从 store + viewport 读状态。
5. **去底与遮罩**:导出前先把框裁进离屏 canvas;如有手工 mask,用 `destination-in` 套 alpha;如启用去底,从裁切块四边 flood-fill 接近目标色的连通区域并设为透明,避免挖掉内部白字/高光。
6. **导出**:对单个框,用离屏 canvas `drawImage(src, x,y,w,h, 0,0,w,h)` 裁出 → `toBlob('image/png')`:
   - **下载**:Blob → object URL → `<a download="<名>.png">` 触发。
   - **复制**:`navigator.clipboard.write([new ClipboardItem({ 'image/png': blob })])`;特性检测,不支持(非 Chromium)则禁用"复制"并提示"复制需 Chrome/Edge"。
   - **下载全部**:逐框下载(浏览器可能弹"允许多文件下载",可接受)。
   - **选择文件夹**:Chrome/Edge 下可用 File System Access API 写入用户选择的目录;目录句柄保存在 IndexedDB。

## 数据模型

框(内存 + localStorage,不导出为文件):
```
{ id: string, name: string, x: number, y: number, w: number, h: number }  // 源图像素
```

实际实现还会为每框保存 `removeWhite/tol/bgColor/mask` 等浏览器本地状态,用于刷新后恢复。导出物只有 PNG 像素,不含任何坐标/元数据文件。

## 错误处理

- 未加载图片:导出/下载类控件禁用,提示"先拖入一张图"。
- 框尺寸 < 3px 或为负:不创建。
- 框越界:裁切时把矩形夹到图像范围内。
- "下载全部"遇重名:自动加 `_2`、`_3` 后缀(单个下载用原名,由 OS 去重)。
- 剪贴板 API 不可用:禁用"复制"并给出原因提示,"下载"不受影响。
- 文件夹写入 API 不可用或权限被拒:回退浏览器默认下载。
- 去底误伤:调低容差、换吸管目标色、关闭去底,或用"还原"画笔把遮罩区域补回。

## 验证

纯手动(无遗留的复杂纯逻辑,无需自动化测试):

1. 浏览器打开页面,拖入仓库已有的扁平 UI 概念图(取自 `docs/visual/exports/ui/2026-06-28_battle-ui-sliced_v01/` 的源)。
2. 画 2~3 个框,移动/缩放/微调/命名/删除各试一遍。
3. 调去底/吸管/容差/画笔/清除遮罩,确认右侧预览等于导出结果。
4. "下载 PNG":确认文件名、透明区域和像素内容符合预期。
5. "复制"(Chrome/Edge):到 Figma Ctrl+V,确认贴入图正确。
6. 刷新页面:确认框、设置和遮罩从 localStorage 恢复。

## 落点 / 与项目约定衔接

- 工具文件:`tools/ui-slicer/index.html`;`package.json` 提供 `npm run ui:slicer` 用 `start` 打开。
- 产出 PNG 是用户的中间素材,直接进 Figma;**不**自动落进 `assets/resources/art/`(沿用项目铁律:上游视觉素材未经审核不得进运行资源)。若日后某些成品要进游戏,另走 `美术资源管线`。
- 这是"人工框选 + 轻量去底/遮罩修边"的素材工具,符合 `UI与美术产出管线` 对上游参考/中间素材的定位。
