# 视频拆帧管线（人机分工版） 设计

> **修订（2026-07-07）**：分工再调整——「循环点裁剪留人工」作废，改由挑帧网页接管（自动循环点检测+弧长推荐，人只点选关键帧）；抠图/锚点按纯侧视假设加固。见 `2026-07-06-sideview-frame-pipeline-design.md`。
> 日期：2026-07-05 ｜ 状态：已与用户对齐
> 范围：把「即梦生视频 → 序列帧入游戏」流程中机械环节自动化：均匀拆帧（拆帧率=播放 fps）、白底抠图、锚点稳定、合规检查、预览页。人工保留生成、剪辑、验收三个判断点。
> 不在范围：即梦 CLI 集成、自动选段算法（前身方案已废弃）、ArtManifest 自动登记、attack/death 铺量（链路验证后重复执行即可）。
> 前身：`2026-07-05-3d-sprite-pipeline-design.md`（3D 渲染路线，风格判负）；其「转向」一节所述"即梦 CLI + 自动选段"方案被本设计取代。

## 决策记录

1. **人机分工由用户定案**：生成质量判断和循环点裁剪最依赖人眼，留给用户（即梦网页生成 + 剪映裁剪最小动作单元，0.5~1 秒）；拆帧/抠图/对齐/合规交给脚本。此分工同时删掉了前身方案里风险最高的两块（即梦 CLI Windows 兼容性未知、自动选段算法待验证）。
2. **拆帧率 = 播放 fps（1:1 时间保真）**：按游戏播放帧率均匀拆帧，不做关键帧挑选，动作节奏与视频真实时间一致。现有 `sprite-keyframe-tool.py` 的均匀选帧模式 + `--sample-fps` 原生支持。
3. **复用不重写**：`sprite-keyframe-tool.py` 的白底抠图/站地锚点/脚尖硬锁/游戏帧高钳制体系成熟，新脚本只做薄编排。
4. 沉淀自 3D 路线验证的资产继续可用：对比预览页模式（棋盘格/游戏主背景真实比例垫底）、`docs/visual/staging/characters/` 目录约定。

## 1. 输入约定（人工）

- 即梦生成角色动作视频：**白底、镜头固定、无运镜**；剪映裁出动作最小单元（循环动作首尾衔接自己掐准）。
- 按动作名放入 `docs/visual/staging/characters/<角色>-video/clips/`，如 `idle.mp4`——**文件名 = 动作键**。
- 剪辑时长约束（2026-07-05 修订，「宁短勿卡」）：帧数红线 10 帧不变，fps 上限提到 **18**（fps 对性能几乎免费，显存只看帧数）；循环单元时长上限 = 10 ÷ fps（**18fps→0.56s**，12fps→0.83s）。超时脚本自动降 fps 保帧数并警告——降到 <12fps 视为卡顿不可接受，应回剪映剪短。

## 2. 编排脚本 `tools/sprite-pipeline/split_action.py`

- 用法：`python tools/sprite-pipeline/split_action.py --clips <目录> [--fps 12] [--action <名>]`（缺 `--action` 时处理目录下全部视频）。
- 流程（每个视频）：
  1. ffprobe 读时长 → 目标帧数 = round(时长 × fps)，>10 帧打红线警告；循环视频按 [0, 时长) 半开区间均匀取样（用户剪的循环单元尾帧≈首帧，含终点会得到重复帧）；
  2. 调 `sprite-keyframe-tool.py`：`--sample-fps <fps>` + 均匀选帧模式 + 目标帧数 + 现有抠图/锚点/脚尖硬锁/游戏帧高默认参数；
  3. 产出到 `temp/sprite-keyframes/pipeline/<角色>/<动作>/`（game_keyframes 规格）；
  4. 生成/更新预览页：每个动作按目标 fps 播放，棋盘格 + 游戏主背景（角色 120px : 屏高 1920px 真实比例）+ 深色底三格。
- 失败处理：视频缺失/ffprobe 失败 → 明确报错跳过该动作；抠图后有效帧不足目标帧数 → 报错并提示视频白底质量问题（不静默凑帧）。

## 3. 验收标准（最小验证 = dps idle 一个动作）

- 用户剪一段 idle 白底视频丢入 clips → 一条命令产出 game_keyframes + 预览页；
- 预览页动画节奏与剪映里播放的视频一致（1:1 时间保真）；
- 抠图边缘、脚点稳定性不劣于现网页版工具手动操作的结果（同一套算法，理应一致）；
- 用户预览认可后按 `ai/skills/美术资源管线.md` 正式导入，游戏内验证。

## 4. 验证通过后的展望（本期不做）

- run/attack/death 及怪物动作按同流程铺量（每动作 = 生成+剪一段视频丢目录）；
- 预览页加"直接生成 ArtManifest 登记草稿"按钮；
- 若某类动作反复生成不满意，再评估即梦 CLI 批量生成候选（前身方案可部分复活）。
