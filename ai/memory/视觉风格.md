# Cgame 视觉风格

> 当前视觉风格、UI 质感和 GPT 生图提示词积木的权威来源。每轮 GPT 网页生图 + Figma 整理后更新这里；历史 prompt 与导出图放 `docs/visual/`。

## 当前状态

- 第一版战斗主界面视觉以用户放入 Figma 的 GPT PNG 为准：`Cgameui` 文件中的 `Battle_UI_Reference`（1080×1920，底图锁定 + 透明热区）。
- 正确 Figma 工作流：不要重绘整张图，也不要把原图改成扁平矢量；保留 raster 原图，做 `Battle_UI_Raster_Slices` 裁切分层和 `Battle_UI_Editable_Trace` 的 UI 组件临摹。
- 项目仍处色块占位阶段；真实美术与 UI 示意先在 Figma/`docs/visual/` 中探索，确认进游戏后再走 `assets/resources/art/` + ArtManifest。
- 视觉方向必须服务现有玩法：竖屏整体，核心战斗横向展开，小队在左、敌人从右进场。

## 风格锚点

- 世界观：轻水墨武侠 / 废墟江湖 / 妖潮入侵，偏放置 RPG 战斗页而非宣传海报。
- 情绪：清雅、明亮、有一点危机感；背景有荒废山门和怪物压境，但不走黑暗末日风。
- 明暗：米白宣纸底为主，浅灰水墨远山和迷雾退后，战斗单位与 UI 用更深轮廓拉清晰度。
- 夸张程度：Q 版小人和怪物，剪影要适合手机小屏识别；Boss 可以明显放大，但不要压住 UI。
- 参考排除：不要重活动入口、充值入口、挂机宝箱、宣传大标题，也不要做成纯插画海报。

## 色彩与材质

- 主色：米白、宣纸黄、浅灰绿墨色。
- 强调色：青玉绿、淡竹绿、暖金、木纹棕；少量朱砂红只用于红点或敌方/危险信息。
- 背景层次：远山、竹林、山门、木塔都低饱和低对比，给角色和 UI 让位。
- 角色/敌人对比：玩家用偏绿/蓝/米色衣装与绿色血条；敌人用深灰黑剪影、红眼、红血条和少量蓝紫能量。
- UI 材质：宣纸浅底、墨边、玉石按钮、铜金描边、轻阴影；资源条可用深灰半透明胶囊。

## 角色 / 怪物规则

- **(2026-07-05) 战斗单位新方向定案：极简水墨 Q 版**——圆头点眼、2~2.5 头身、毛笔墨线（粗细顿挫）+ 淡彩大色块（宣纸米白/青玉绿/淡竹绿），细节克制、部件（头/发/身/武器）轮廓互不缠绕（服务 PartsRig 拆件）。首个过审形象 `docs/visual/staging/characters/new_dps.png`；生成提示词见该日对话沉淀（白衣角色**用浅灰底生成**，避免白吃白抠图事故）。旧精致 Q 版立绘风格（长发丝/多挂饰）退役，小队其他角色与怪物按新方向统一。

- 角色剪影：三名玩家必须在左侧同一水平战斗线上，轮廓分别区分草帽侠客、绿衣女侠、蓝灰少年剑客。
- 怪物剪影：小怪和 Boss 在右侧同一水平战斗线上；Boss 体型更大，可带蓝紫/青色能量边缘。
- 阵营区分：玩家在左、敌人在右；玩家血条绿色，敌人血条红色。
- 小尺寸可读性：先保证站位、比例、轮廓和血条清楚，细节可以后置。
- 动画/序列帧注意：未来接游戏资源时优先做 idle/attack 的少帧序列；脚底基准线必须统一，避免战斗中上下跳线。

## UI 规则

- 战斗主界面：1080×1920 竖屏；顶部 HUD 约 0–260，中部战斗区约 260–1450，技能区约 1450–1640，底部导航约 1640–1920。
- 装备/背包：后续沿用宣纸面板 + 墨边/金边 + 品质色；不要做成网页后台。
- 结算/奖励：后续应短文案、强奖励物图标，避免大段说明。
- 按钮与图标：技能按钮圆形玉石/金属描边；底部 6 导航，战斗项放大高亮，图标走古风可读剪影。
- 信息密度：手机小屏优先；顶部只保留头像、战力、资源、关卡标题、波次进度和目标奖励卡；底部只保留 3 技能 + 6 导航。

## 禁忌

- 不要：首充、七日礼、江湖令、活动、挂机奖励宝箱。
- 不要：暂停、加速、自动、跳过按钮。
- 不要：左侧竖排功能按钮、大弹窗、大段说明文字、宣传标题。
- 不要：战斗单位上下错位；玩家和怪物必须脚底同线。
- 不要：大面积黑色末日风、过复杂背景、过度写实导致小屏不可读。

### 风格前缀

```text
轻水墨武侠、轻古风、Q版放置RPG、竖屏微信小游戏实际战斗界面，米白宣纸底，浅灰水墨山景，淡竹绿和青玉绿点缀，暖金和木纹棕 UI 描边，纸张、墨边、玉石、铜金质感，背景淡、角色和 UI 清晰，手机小屏可读。
```

### 负向约束

```text
不要宣传海报，不要大标题，不要首充/七日礼/江湖令/活动/挂机奖励，不要暂停/加速/自动/跳过，不要左侧竖排功能按钮，不要大弹窗，不要大段文字，不要黑暗末日风，不要过度写实，不要背景抢主体，不要让战斗单位上下错位。
```

### 常用画幅

- 竖屏 UI：9:16。
- 横向战斗背景：16:9 或按当前 Cocos 视口裁切。
- 独立图标/资产：透明背景，方图或按目标资源比例。
