## 一句话

竖屏整体、**核心战斗横向展开**的多人小队自动战斗微信小游戏（Cocos 3.8.8，`F:\Cgame`）。

## 已完成

- 战斗循环：当前为 1 人小队（仅 dps，默认近战）；历史 tank/dps/healer 配置仍保留，敌人从右进场、无碰撞群殴。
- 战斗框架：统一 `stats` 属性表 + 唯一减法伤害公式；维度有 暴击/暴伤(加成式)/格挡/闪避/攻速/伤害加成/伤害减免/射程。
- 关卡系统：`enemyTypes` 怪物类型表（丧尸/疾行者/重装）+ `levels` 关卡表（每波可混多种怪），打完该关所有波即通关；结算页支持下一关/背包/重打，进度存档记录当前关与最高解锁。
- 表现：干净战斗背景图 + 还原稿主界面 UI 已接入；Figma 加载页与 `01 Begin` 开始页切片已接进 BootFlow；小队角色已有 idle/run/attack/death 动作状态与临时序列帧，怪物/子弹仍是色块占位 + 战斗飘字。
- 工具：游戏内实时调参面板（`ConfigPanel`，临时手感调试）+ 默认折叠隐藏的动作预览面板（`ActionPreviewPanel`，选任意 `ArtManifest` 序列帧键预览游戏内 SpriteFrame/.meta/缩放效果）；**Excel 导表管线**；本地 `ui-slicer`；本地 `sprite-keyframe`（按起止秒截视频片段 + 抠白底 + 站地/身体锚点稳定 + 视觉锁定 + 脚尖硬锁 → `game_keyframes` 游戏导入预览 → 序列帧导入）。
- **UI 与美术产出管线**（上游协作）：GPT 网页生图/提示词 → Figma 整理 → `ai/memory/视觉风格.md` 风格沉淀 → `docs/visual/` 保存参考、提示词、UI/美术示意图；当前 `Cgameui` 已有 `01 Begin` 开始页稿。
- **美术资源管线**（`art/` + `resources/art/`）：逻辑键 + 缺图回退色块；ArtManifest 登记 + check-art 校验 + 序列帧动画；接美术不动玩法代码。
- **装备系统**：背包/仓库/每角色 5 装备栏 + 掉落/转移/穿脱 + 锁定/排序/出售/批量出售 + 持久化 + 点击详情/差值对比 + 列表滑动 + 拖拽投放；装备属性由 `equip.xlsx` 管理（部位基础、品质倍率、roll、附加词条池），胜利掉落由 `battle.xlsx` 的 `dropGroup` 指向 `drop.xlsx` 权重配置，穿戴经 `EffectiveStats` 接进战斗。
- **装备等级 + 合成**：所有装备携带等级（与品质独立叠乘），玩家用材料（打造石/宝石碎片/铭文粉尘）选部位+档位合成装备，材料够就必出装备（等级/品质按档位配置随机）；`craft.xlsx` 管理档位材料成本与品质权重。
- **服务层与宝箱**：seed 随机、统一奖励包、胜利奖励服务、装备服务包装、宝箱库存上限、在线击杀/关底宝箱掉落、离线快速模拟同规则、宝箱选择与开箱奖励展示已落地；离线不直接产装备，开箱再把宝箱转成装备/材料且不给金币经验。
