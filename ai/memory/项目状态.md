## 最近进展（给新对话）

> 这是新对话的第一落点。每次开发收尾时刷新成 3~6 行：刚做完什么、下一步、有什么坑。

- 当前出战阵容为 1 人小队，只生成默认近战 `dps`；配置真源是 `tools/config-xlsx/battle.xlsx` 的 `Misc.roster=dps`，tank/dps/healer 定义仍保留。
- 角色动作逻辑已接好：`BattleManager` 输出 `idle/run/attack/death`，`BattleEntry` 按 `char/<职业>/<动作>` 播放序列帧；新增网页预览内的“动作预览”面板，可直接选 `ArtManifest` 里的任意序列帧键看游戏内播放效果。
- 刚定位 idle 微偏移：视频本身稳定，但抠图后的透明 PNG 内容包围盒已横向漂移约 13.5px、底部约 2px，所以问题发生在“视频帧 → 抠图 PNG”的锚点稳定链路，不是 Cocos 播放层。
- `sprite-keyframe` 工具的默认 `anchor-mode=foot` 已改为“站地稳定”：身体核心锁横向、脚底落地线锁纵向；网页结果区现在会生成/播放 `game_keyframes/`，与导入到 Cocos 的共同裁切 PNG 保持一致。
- 启动流程仍是 `加载页 -> 开始页 -> 主界面/战斗`；Figma `Cgameui` 的加载页与 `01 Begin` 开始页都已切片进 `assets/resources/art/ui/boot/`，并补 full-frame custom `.meta`，`BattleEntry` 先显示加载墨圈/进度条，资源就绪后切到开始页。
- 延续坑点：新增动作 PNG 的目录 `.meta` 可能由 Cocos 导入生成，若序列帧 `.png.meta` 恢复成 `trimType: "auto"`，要用序列帧工具重新导入或修补为 full-frame custom，避免角色帧间抖动/忽胖忽瘦；当前 `char/dps/idle` 已统一修成 custom。

## 一句话

竖屏整体、**核心战斗横向展开**的多人小队自动战斗微信小游戏（Cocos 3.8.8，`F:\Cgame`）。

## 已完成

- 战斗循环：当前为 1 人小队（仅 dps，默认近战）；历史 tank/dps/healer 配置仍保留，敌人从右进场、无碰撞群殴。
- 战斗框架：统一 `stats` 属性表 + 唯一减法伤害公式；维度有 暴击/暴伤(加成式)/格挡/闪避/攻速/伤害加成/伤害减免/射程。
- 关卡系统：`enemyTypes` 怪物类型表（丧尸/疾行者/重装）+ `levels` 关卡表（每波可混多种怪），打完该关所有波即通关；结算页支持下一关/背包/重打，进度存档记录当前关与最高解锁。
- 表现：干净战斗背景图 + 还原稿主界面 UI 已接入；Figma 加载页与 `01 Begin` 开始页切片已接进 BootFlow；小队角色已有 idle/run/attack/death 动作状态与临时序列帧，怪物/子弹仍是色块占位 + 战斗飘字。
- 工具：游戏内实时调参面板（`ConfigPanel`，临时手感调试）+ 动作预览面板（`ActionPreviewPanel`，选任意 `ArtManifest` 序列帧键预览游戏内 SpriteFrame/.meta/缩放效果）；**Excel 导表管线**（`tools/`，多源驱动 + `npm run config` + 校验 + E2E）；本地 `ui-slicer`；本地 `sprite-keyframe`（抠白底 + 视频拆关键帧 → 站地/身体锚点稳定 → `game_keyframes` 游戏导入预览 → 序列帧导入）。
- **UI 与美术产出管线**（上游协作）：GPT 网页生图/提示词 → Figma 整理 → `ai/memory/视觉风格.md` 风格沉淀 → `docs/visual/` 保存参考、提示词、UI/美术示意图；当前 `Cgameui` 已有 `01 Begin` 开始页稿。
- **美术资源管线**（`art/` + `resources/art/`）：逻辑键 + 缺图回退色块；ArtManifest 登记 + check-art 校验 + 序列帧动画；接美术不动玩法代码。
- **装备系统**：背包/仓库/每角色 5 装备栏 + 掉落/转移/穿脱 + 持久化 + 点击详情/差值对比 + 列表滑动 + 拖拽投放；装备属性由 `equip.xlsx` 管理（部位基础、品质倍率、roll、附加词条池），胜利掉落由 `battle.xlsx` 的 `dropGroup` 指向 `drop.xlsx` 权重配置，穿戴经 `EffectiveStats` 接进战斗。
