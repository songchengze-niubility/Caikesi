# 代码地图

> 模块 → 职责 → 关键文件。**新增/移动模块或改了职责时必须同步更新这里**，否则导航失真。
> 最后更新：2026-07-04。

## 入口

- **场景** `assets/main.scene`：唯一战斗场景，名为 `main`。
- **挂载脚本** `assets/scripts/BattleEntry.ts`：挂在场景的节点上，是当前唯一需要在编辑器挂载的脚本。

## 战斗核心（玩法逻辑）

| 文件 | 职责 | 改这里当… |
|------|------|-----------|
| `combat/BattleManager.ts` | 纯逻辑：布阵、刷怪、移动、攻击、命中、治疗、胜负，维护单位 `idle/run/attack/death` 动作状态，并吐出 `enemyKilled` 事件。不碰渲染/存档 | 改战斗行为/流程/动作触发/击杀事件 |
| `combat/EffectiveStats.ts` | 装备 → 战斗属性的纯计算层：基础 `CombatStats` + 已穿装备 `stats`，并做概率/生命/攻速钳制 | 改装备如何影响战斗 |
| `combat/CombatFormula.ts` | **唯一伤害结算入口** `calcDamage(att,def)`：闪避→减法→暴击→格挡→加成→减免 | 改伤害公式/加结算维度 |

## 配置（数值由 Excel 管理，策划改 Excel）

> 改战斗/关卡数值：编辑 `tools/config-xlsx/battle.xlsx` → `npm run config` → 生成 `battle.config.generated.ts`；当前出战阵容可由 `battle.xlsx` 的 `Misc.roster` 覆盖（如 `dps`）。改装备数值：编辑 `tools/config-xlsx/equip.xlsx` → 生成 `equip.config.generated.ts`；改装备掉落：编辑 `tools/config-xlsx/drop.xlsx` → 生成 `drop.config.generated.ts`；改宝箱掉落：编辑 `tools/config-xlsx/chest.xlsx` → 生成 `chest.config.generated.ts`；改离线金币/经验效率：编辑 `tools/config-xlsx/offline.xlsx` → 生成 `offline.config.generated.ts`。改合成配方：编辑 `tools/config-xlsx/craft.xlsx` → 生成 `craft.config.generated.ts`。详见 `ai/skills/配置与关卡.md`。

## 装备系统（`inventory/` 存储/UI + `EffectiveStats` 接战斗）

| 文件 | 职责 |
|------|------|
| `inventory/InventoryModel.ts` | 纯逻辑（**不依赖 cc**，可 tsx 单测）：背包/仓库 + **每角色一套 5 装备栏**(`equipped[角色][部位]`) + 掉落、转移、穿脱、锁定、排序、出售、批量出售白绿 + 序列化/读档迁移（旧装备缺 `stats` 时自动补属性）。操作返回 `OpResult`/`SellResult`，满了/非法即失败不留半成品 |
| `inventory/EquipDefs.ts` | 部位/品质枚举 + 颜色 + 名字占位池 + `EquipItem.locked` + `createEquipItem(slot, quality)`/`randomItem()`；新掉落从 `EquipConfig` 生成带 roll/附加词条的 `stats`，并提供 UI 属性格式化 |
| `inventory/InventoryService.ts` | 纯逻辑服务包装：装备加入、锁定、出售、批量出售、排序入口；战斗/界面以外的系统走这里，不直接碰 UI |
| `inventory/EquipmentService.ts` | 纯逻辑服务包装：穿戴/卸下/同部位属性差值对比 |
| `inventory/InventoryConfig.ts` | 小常量：背包/仓库格子上限 |
| `inventory/InventoryView.ts` | Cocos 占位 UI（覆盖层）：**角色切换** + 当前角色 5 栏 + 背包/仓库滑动列表 + 选中装备完整属性/同部位差值/售价 + 锁定/解锁、出售、排序、批售白绿按钮 + 选中后拖拽投放；唯一 import cc 的装备文件 |
| `inventory/InventoryPersistence.ts` | 持久化胶水：`loadInventory/saveInventory` 经 `DataService`（缓存 PlayerData） |
| `tools/inventory-test.ts` | InventoryModel 单测（tsx），`npm run test:inventory`，覆盖胜利奖励用仓库掉落兜底、直接仓库穿戴、装备脱到仓库、老存档补属性、锁定持久化、出售/批量出售、排序 |
| `tools/effective-stats-test.ts` | 装备属性叠算单测（tsx），`npm run test:effective` |
| `tools/drop-config-test.ts` | 掉落配置单测（tsx），`npm run test:drop`，覆盖关卡 `dropGroup` 指针、完整权重和按权重生成装备 |
| `craft/CraftService.ts` | 材料合成装备纯逻辑：`craftEquipment(materials, tierId, slot, rng)`，材料够就产出装备（等级/品质按 `CraftConfig` 档位随机），不够/非法参数返回失败且不改材料 |

> 持久化复用 `core/data/DataService.ts`（`PlayerData.inventory?` 字段）。读档会给老存档缺 `stats` 的装备补属性；直接构造的异常无 `stats` 装备在 `EffectiveStats` 层仍按 0 加成处理。
