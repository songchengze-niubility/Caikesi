# 代码地图

> 模块 → 职责 → 关键文件。**新增/移动模块或改了职责时必须同步更新这里**，否则导航失真。
> 最后更新：2026-07-07。

## 入口

- **场景** `assets/main.scene`：唯一战斗场景，名为 `main`。
- **挂载脚本** `assets/scripts/BattleEntry.ts`：挂在场景的节点上，是当前唯一需要在编辑器挂载的脚本。

## 战斗核心（玩法逻辑）

| 文件 | 职责 | 改这里当… |
|------|------|-----------|
| `combat/BattleManager.ts` | 纯逻辑：布阵（构造函数第5参 `roster` 决定出战单位，缺省回退 `BattleConfig.roster`）、刷怪、移动、攻击、命中、治疗、胜负，维护单位 `idle/run/attack/death` 动作状态，吐出 `enemyKilled`/`skillCast` 事件；自动技能两个钩子（每帧 tick / 普攻挥出计数）。不碰渲染/存档 | 改战斗行为/流程/动作触发/战斗事件 |
| `combat/EffectiveStats.ts` | 装备 → 战斗属性的纯计算层：基础 `CombatStats` + 已穿装备 `stats` + **镶嵌加成**（`itemInlayStats`），并做概率/生命/攻速钳制；`buildEffectiveStatsMap` 第二参 `levels?` 可选，按角色等级放大 hp/atk 后再叠装备，不传时行为与旧版字节级等价（`sim:pacing` 不传，不受影响） | 改装备如何影响战斗 / 改等级如何影响战斗 / 改镶嵌如何影响战斗 |
| `combat/CombatFormula.ts` | **唯一伤害结算入口** `calcDamage(att,def)`：闪避→减法→暴击→格挡→加成→减免 | 改伤害公式/加结算维度 |
| `combat/SkillRuntime.ts` | 自动技能纯逻辑（不依赖 cc）：`UnitSkills` 计时/计数触发、无目标保留待放、aoe/nearest/single 目标选择；挂单位实例（Boss 技能可复用）；伤害结算由 BattleManager 走 calcDamage×倍率 | 改技能触发/目标选择/加技能形态 |

## 配置（数值由 Excel 管理，策划改 Excel）

> 改战斗/关卡数值：编辑 `tools/config-xlsx/battle.xlsx` → `npm run config` → 生成 `battle.config.generated.ts`；当前出战阵容可由 `battle.xlsx` 的 `Misc.roster` 覆盖（如 `dps`）。改装备数值：编辑 `tools/config-xlsx/equip.xlsx` → 生成 `equip.config.generated.ts`；改装备掉落：编辑 `tools/config-xlsx/drop.xlsx` → 生成 `drop.config.generated.ts`；改宝箱掉落：编辑 `tools/config-xlsx/chest.xlsx` → 生成 `chest.config.generated.ts`；改离线金币/经验效率：编辑 `tools/config-xlsx/offline.xlsx` → 生成 `offline.config.generated.ts`。改合成配方：编辑 `tools/config-xlsx/craft.xlsx` → 生成 `craft.config.generated.ts`。详见 `ai/skills/配置与关卡.md`。

## 角色成长（`growth/`，纯逻辑 + 战斗/持久化接线）

| 文件 | 职责 |
|------|------|
| `growth/CharGrowthConfig.ts` | 纯计算（不依赖 cc）：`charLevelCoef(level)` 等级系数（镜像 `EquipConfig.levelCoefficient`）、`expToNext(level)` 经验曲线（几何增长）、`clampCharLevel(level)` 钳制 1~`maxLevel`；数值来自 `BattleConfig.charGrowth` |

| `tools/char-growth-test.ts` | CharacterGrowthModel 单测（tsx），`npm run test:growth`，8 项覆盖连续升级/封顶/序列化/自愈 |

> 击杀经验来源：`BattleConfig.enemyTypes[type].exp`（`EnemyTypes` 表新增列）。`BattleEntry` 在 `_processBattleEvents` 消费 `enemyKilled` 累计 `_battleExpGained`；战斗结束（胜或负，`update()` 的 `phaseBefore` 边缘触发）调用 `_commitBattleExp()`，给**全体上阵角色各一份全额**（非均分、下阵不得），经 `CharacterGrowthModel.gainExp` 落库并 `saveGrowth`；离线收益（`OfflineClaimService`）同样喂养上阵角色，不再写全局 `PlayerData.exp`。`_startBattle` 把 `this._growth.levelOf(cls)` 组成的等级表传给 `buildEffectiveStatsMap(equipped, levels)`，只放大 hp/atk。**本次只做机制，不重调平衡**：`pacing-sim` 不传 `levels`，13 门槛保持"纯装备档位"全绿；角色等级会让现有关卡随练度变简单，列为待办（后续折进 `pacing-sim` 的 LOADOUT 逐关重调）。设计/计划见 `docs/superpowers/{specs,plans}/2026-07-05-character-exp-level*`。
