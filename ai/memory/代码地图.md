# 代码地图

> 模块 → 职责 → 关键文件。默认用 `rg` 搜关键词后只读相关章节，不属于每次会话的全文必读项。**新增/移动模块或改了职责时必须同步更新这里**，否则导航失真。
> 最后更新：2026-07-11。

## 入口

- **场景** `assets/main.scene`：唯一战斗场景，名为 `main`。
- **挂载脚本** `assets/scripts/BattleEntry.ts`：当前唯一需要在编辑器挂载的脚本；是组合根，负责生命周期、读档、战斗会话与跨模块编排，不直接承载业务面板和战场渲染。

## 战斗核心（玩法逻辑）

| 文件 | 职责 | 改这里当… |
|------|------|-----------|
| `combat/BattleManager.ts` | 纯逻辑：布阵（构造函数第5参 `roster` 决定出战单位，缺省回退 `BattleConfig.roster`）、刷怪、移动、攻击、命中、治疗、胜负，维护单位动作状态与 Buff 帧逻辑（`_updateBuffs` 周期跳伤/到期）；`Projectile` 弹道（直线/穿透/抛物，`MAX_PROJECTILES=64`）与 `ZoneEffect` 场地（周期施加效果列表，`MAX_ZONES=8`）；嘲讽拉仇恨/沉默禁技/击退钳制落地；**行军推进**（清波→`marching` 按最慢 stats.moveSpeed 走 `Wave.distance`→刷下一波，末波清完即胜；表现假象不动局部坐标）。事件：`enemyKilled`/`skillCast`/`buffChanged`/`zoneSpawned`/`zoneExpired`/`knockback`/`marchStarted`/`marchEnded`。伤害/治疗/上Buff 一律经 `Effects.applyEffect`，不碰渲染/存档 | 改战斗行为/流程/弹道/场地/行军/战斗事件 |
| `combat/CombatUnit.ts` | **统一单位模型**（士兵/敌人共用）：side/key/archetype/attackInterval/cd 归一字段（移速在 `stats.moveSpeed`）+ 双方工厂 + `recomputeDerived` 脏重算。stats 双引用约定：无 buff 时 `stats===baseStats` 同一引用（ConfigPanel 实时调参依赖），有 buff 时为聚合副本 | 加单位字段 / 改单位构造 / 加原型（Boss） |
| `combat/Effects.ts` | **唯一状态变更入口** `applyEffect(source,target,effect,hooks,floatKind?,tags?)`：damage（走 calcDamage 并透传伤害标签）/heal/applyBuff（快照 `srcMult=1+全伤害+技能伤害` 供 DoT）/dispel（knockback/summon 第2/3段占位）。普攻/子弹/敌攻/技能全走它；加内容=加 Effect 数据不改管线 | 加效果种类 / 改状态变更行为 |
| `combat/BuffSystem.ts` | Buff 纯函数：`applyBuffStack(…,srcMult?)`（refresh/add 叠层）/`dispelByTag`/`tickBuffs`（周期+到期 swap-remove）/`buffedStats`（flat/pct×层数聚合+钳制）/`buffGate`（stun 行为门）；实例含 srcAtk+srcMult 快照供 DoT 结算（跳伤×srcMult，HoT 不乘） | 改 Buff 生命周期/叠层/属性聚合规则 |
| `combat/EffectiveStats.ts` | 装备 → 战斗属性的纯计算层，**双层公式（2026-07-11）`面板=(白板+固定)×(1+百分比合计)`**：三围/移速百分比键（hpPct 等）经 `PCT_MAP` 聚合；`calcEffectiveStats(base,items,levelPct?)` 第三参为角色等级三围百分比（乘全池含 def，不作用 moveSpeed）；概率/生命/攻速钳制（`normalizeStats` 含 18 维，供 BuffSystem 复用）；`buildEffectiveStatsMap` 的 `levels?` 不传时与旧行为等价（`sim:pacing` 不传，不受影响） | 改装备如何影响战斗 / 改等级如何影响战斗 / 改镶嵌如何影响战斗 |
| `combat/CombatFormula.ts` | **唯一伤害公式** `calcDamage(att,def,tags?)`：闪避→减法→暴击→格挡→加成→减免；**伤害标签**（来源 basic/skill × 范围 single/aoe）同乘区加算：×(1+全伤害+(普攻\|技能)+(单体\|群体))，无标签只吃全伤害 | 改伤害公式/加结算维度/改标签语义 |
| `combat/SkillRuntime.ts` | 自动技能纯逻辑（不依赖 cc）：`UnitSkills` 计时/计数触发、**技能急速 `haste`**（计时 ×(1+急速)；计数需求 ÷(1+急速) 向上取整保底 1，BattleManager 每帧用面板 skillHaste 刷新）、无目标保留待放、aoe/nearest/single 目标选择；挂单位实例（Boss 技能可复用）；结算由 BattleManager 按 `SkillDef.effects` 效果列表逐个走 applyEffect | 改技能触发/目标选择/加技能形态/改急速 |
| `combat/PassiveSystem.ts` | 被动纯逻辑（注入 rng 可测）：`firePassives`（钩子过滤+chance 掷骰+targetMode 解析 trigger/self/team）、`applyAlwaysPassives`（开战常驻/光环）。钩子位点在 BattleManager：普攻命中 onHit/直击受击 onHurt/直击击杀 onKill（Projectile/Zone 带 owner 溯源）/放技能 onCast；`_procGuard` 一层防递归 | 加被动形态/改触发语义 |

## 配置（数值由 Excel 管理，策划改 Excel）

> 改战斗/关卡数值：编辑 `tools/config-xlsx/battle.xlsx` → `npm run config` → 生成 `battle.config.generated.ts`；当前出战阵容可由 `battle.xlsx` 的 `Misc.roster` 覆盖（如 `dps`）。改装备数值：编辑 `tools/config-xlsx/equip.xlsx` → 生成 `equip.config.generated.ts`；改装备掉落：编辑 `tools/config-xlsx/drop.xlsx` → 生成 `drop.config.generated.ts`；改宝箱掉落：编辑 `tools/config-xlsx/chest.xlsx` → 生成 `chest.config.generated.ts`；改离线金币/经验效率：编辑 `tools/config-xlsx/offline.xlsx` → 生成 `offline.config.generated.ts`。改合成配方：编辑 `tools/config-xlsx/craft.xlsx` → 生成 `craft.config.generated.ts`。详见 `ai/skills/配置与关卡.md`。
