# 代码地图

> 模块 → 职责 → 关键文件。**新增/移动模块或改了职责时必须同步更新这里**，否则导航失真。
> 最后更新：2026-06-26。

## 入口

- **场景** `assets/main.scene`：唯一战斗场景，名为 `main`。
- **挂载脚本** `assets/scripts/BattleEntry.ts`：挂在场景的节点上，是当前唯一需要在编辑器挂载的脚本。

## 战斗核心（玩法逻辑）

| 文件 | 职责 | 改这里当… |
|------|------|-----------|
| `combat/BattleManager.ts` | 纯逻辑：布阵、刷怪、移动、攻击、命中、治疗、胜负。不碰渲染 | 改战斗行为/流程 |
| `combat/CombatFormula.ts` | **唯一伤害结算入口** `calcDamage(att,def)`：闪避→减法→暴击→格挡→加成→减免 | 改伤害公式/加结算维度 |

## 配置（数值由 Excel 管理，策划改 Excel）

> 改数值流程：编辑 `tools/config-xlsx/battle.xlsx` → `npm run config` → 生成 `battle.config.generated.ts`。详见 `ai/skills/配置与关卡.md`。

| 文件 | 职责 |
|------|------|
| `config/BattleConfig.ts` | **只剩类型定义** + `SoldierClass` 联合类型；数值对象 `BattleConfig` 引用生成产物 |
| `config/battle.config.generated.ts` | 导表产物（勿手改），结构 = 历史手写版；BattleManager/ConfigPanel 引用它 |
| `tools/excel-to-config.ts` | 导表器：读 6 sheet → 校验（缺字段/引用/键一致/跳号报错）→ 生成产物。`npm run config` |
| `tools/config-xlsx/battle.xlsx` | **数值真源**：Stats/EnemyTypes/Classes/Levels/Misc/Scene 六表 |
| `tools/seed-battle-xlsx.ts` | 从零重建 xlsx 的种子脚本（极少用）；`tools/_e2e-test.ts` 回环自测 |

## 渲染与调试

| 文件 | 职责 |
|------|------|
| `BattleEntry.ts` | 驱动 BattleManager + Graphics 画色块/飘字；建背景节点；接美术资源管线（有图用 Sprite/序列帧，缺则色块）；点击重开 |
| `combat/Background.ts` | 背景占位：多层视差滚动；`setUsingSprite(true)` 后停渐变重画（让位真实背景图） |
| `debug/ConfigPanel.ts` | 游戏内 HTML 调参面板（仅网页预览，临时手感调试，可导出 JSON） |

## 美术资源管线（`art/`，逻辑键 + 缺图回退；详见 `ai/skills/美术资源管线.md`）

> 代码只认逻辑键，缺图回退色块。Codex 流程：丢图到 `assets/resources/art/` + 在 ArtManifest 登记 → 人开编辑器导入生成 `.meta` → 游戏自动用上。

| 文件 | 职责 |
|------|------|
| `art/ArtManifest.ts` | **逻辑键→资源描述**的唯一权威（Codex 维护）+ `entryFiles` 路径助手。不依赖 cc |
| `art/ArtRegistry.ts` | 加载/缓存/回退（泛型 + 注入 loader，不依赖 cc）：`preload/getSprite/getFrames/missingKeys` |
| `art/FrameClock.ts` | 序列帧帧下标纯函数 `frameAt`（不依赖 cc） |
| `art/CocosArtLoader.ts` | cc 胶水：`resources.load` 实现真实 loader → `createArtRegistry()` |
| `art/FrameAnim.ts` | cc 胶水：`FrameAnimPlayer` 代码驱动切帧 |
| `assets/resources/art/` | 资源目录（现为占位纯色 PNG：bg/main + char/*/idle 序列帧） |
| `tools/check-art.ts` | `npm run check:art` 校验 ArtManifest 每个文件在磁盘存在；`tools/art-test.ts` 纯逻辑单测 |

## 规模化约定（代码长大时）

- 新增系统（如 装备/存档/UI/网络）：在对应区新建子目录（如 `equip/`、`ui/`），并在本表加一行 + 必要时加一个 `ai/skills/<领域>.md`。
- `combat/` 文件多了再按子系统拆（如 `combat/spawn/`、`combat/ai/`）。
