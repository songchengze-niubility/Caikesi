# Caikesi AI 工作区

智能体的项目记忆与通用技能。入口规则见根目录 `CLAUDE.md`（每次会话强制先读本区的固定小集合，技能按需读）。

## 结构

```
ai/
├── README.md            本文件：地图 + 技能索引
├── memory/              项目记忆（会变，随项目演进更新）
│   ├── 项目状态.md       当前进度 / 方向 / 待办（顶部"最近进展"是新对话接力点）
│   ├── 代码地图.md       模块 → 职责 → 关键文件（导航）
│   ├── 设计日志.md       历史决策 + 理由（"为什么"的权威来源）
│   └── 归档/             已完结里程碑的旧决策（防主日志膨胀）
└── skills/              通用技能（稳定 how-to，按需读）
```

## 技能索引（按任务挑相关的读，别全量加载）

| 技能 | 何时读 | 说明 |
|------|--------|------|
| `skills/工作流程.md` | 每次开工与提交前 | 读取顺序、更新记忆、预览验证、提交规范、边界 |
| `skills/战斗框架.md` | 改战斗逻辑/属性/公式时 | 架构分工、统一 stats、伤害公式、加新维度流程、不变量 |
| `skills/渲染与美术替换.md` | 改渲染/背景/接美术时 | 色块占位、背景视差、换 Sprite/动画/特效、性能 |
| `skills/配置与关卡.md` | 改数值/加怪/编关卡时 | BattleConfig 结构、怪物类型、关卡、调参面板 |
| `skills/性能约束.md` | 新增系统、写每帧逻辑、铺量、接美术、做性能评审时 | 微信小游戏/Cocos 性能底线：热路径、池化、渲染、资源、UI、真机验证 |
| `skills/开发收尾.md` | 开发告一段落/交接/提交前 | 更新记忆、归档设计日志、刷新交接摘要、提交 |

> 新增技能时：在 `skills/` 加文件并补 frontmatter（name/description/when），并在本表加一行。技能多了（>15）按领域分子文件夹（combat/ ui/ data/ ...）。

## 记忆 vs 技能

- **记忆**：随项目变的事实——做到哪、当前方向、为什么这么决定、代码在哪。改了项目就更新。
- **技能**：相对稳定的做法——怎么改配置、加属性维度、换美术、验证提交。除非约定本身变了才更新。

## 信息分工

| 想知道 | 看哪 |
|--------|------|
| 数值/字段含义 | `assets/scripts/config/BattleConfig.ts` 注释 |
| 改了什么、何时 | git 提交记录 |
| 为什么这么设计 | `ai/memory/设计日志.md` |
| 当前进度/待办 | `ai/memory/项目状态.md` |
| 某模块在哪、负责啥 | `ai/memory/代码地图.md` |
| 怎么做某类改动 | `ai/skills/*.md`（按索引挑） |
