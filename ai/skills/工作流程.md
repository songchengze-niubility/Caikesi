# 技能 · 工作流程

## 开工前
1. 读 `ai/README.md` → `ai/memory/项目状态.md` → `ai/memory/设计日志.md` → 相关 skill。
2. 改动类任务先说：目标、影响范围、验证方式。

## 改完后
- **方向性决策** → 追加一条到 `ai/memory/设计日志.md`（带日期+理由）。
- **进度/待办变化** → 更新 `ai/memory/项目状态.md`。
- 数值字段含义写在 `BattleConfig.ts` 注释里，不在记忆里重复。

## 验证（无单测，靠预览）
- Cocos 编辑器 ▶ 预览，看战斗是否符合预期；有报错看控制台。
- 调手感用游戏内调参面板（右上角，仅网页预览）。

## 提交
- 只在用户要求时 `git commit` / `push`。
- 中文 commit message，结尾：`Co-Authored-By: Claude <noreply@anthropic.com>`。
- 本仓库已配 git 身份（farrell / 用户邮箱）。

## 边界
- 不在未确认影响范围时大改框架层（`BattleManager`/`CombatFormula`）。
- 占位阶段不引入真实美术资源；保持逻辑与渲染分离。
