# 技能 · 渲染与美术替换

> **接真实美术现在走「美术资源管线」**（逻辑键 + 缺图回退）——加/换图的具体操作见 `ai/skills/美术资源管线.md`。本文只讲渲染层现状与接入位置。

## 现状：色块占位
全部用 `cc.Graphics` 每帧重画，无真实美术（仅占位纯色 PNG 样板）：
- 士兵=方块（按职业上色）、怪=圆（按类型上色/体型）、子弹=黄点、近战=白线、治疗=绿线、飘字=伤害/暴击/格挡/闪避。
- 渲染都在 `BattleEntry.ts`；与逻辑（`BattleManager`）完全分离。
- **背景与角色已接资源管线**：`BattleEntry` 进场前 `ArtRegistry.preload(...)`，拿到 SpriteFrame 就用 Sprite/序列帧、拿不到回退色块（见下）。

## 背景：可替换的多层视差
`combat/Background.ts` 在独立的 `bgNode`（最底层）画：天空渐变 → 远山(慢) → 云(按大小分速度) → 地面 → 地面纹理(快滚)。参数在 `BattleConfig.scene`。

## 换真实美术的路径（不动战斗逻辑，全走资源管线）
加图不是手动拖 Sprite，而是**丢图 + 在 ArtManifest 登记逻辑键**（操作见 `美术资源管线.md`）；渲染层早已按逻辑键取资源，缺图回退色块。各实体接入位置：
- **背景**（已接）：逻辑键 `bg/main`；有图给 `bgNode` 挂 Sprite 并 `Background.setUsingSprite(true)`（停渐变重画），无图走原渐变。
- **角色**（已接）：逻辑键 `char/<职业>/idle` 序列帧；有则 Sprite+`FrameAnimPlayer`，无则色块方块（血条照画）。
- **怪/子弹/UI**（待接）：按同模板——加逻辑键 + 在 `_render` 对应处「有图用 Sprite、无图色块」。
- **特效**：近战白线、飘字后续换打击特效/动画（同样走管线）。

## 原则
- 只动渲染层，`BattleManager` 数值与玩法不受影响。
- 代码只认逻辑键、不写资源路径；缺图永不报错（回退色块）。
- 先用色块把玩法和手感调对，再上美术（最省成本）。

## 性能备注
背景渐变每帧重画（~48 次 fill）：接真实背景图后 `setUsingSprite(true)` 自动停掉，归零。其余占位 Graphics 同理——接美术后该实体不再走色块分支。
