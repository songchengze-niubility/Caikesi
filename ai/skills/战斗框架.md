# 技能 · 战斗框架

## 文件分工
- `config/BattleConfig.ts` —— 所有数值/配置（策划改这里）；Buff 在 `buff.xlsx`→`BuffConfig`，Effect 编码在 `config/EffectTypes.ts`。
- `combat/BattleManager.ts` —— 纯逻辑：布阵、刷怪、移动、攻击、命中、胜负、Buff 帧逻辑。不碰渲染。
- `combat/CombatUnit.ts` —— 统一单位模型（士兵/敌人共用）+ 工厂 + `recomputeDerived` 脏重算。
- `combat/CombatFormula.ts` —— **唯一伤害公式** `calcDamage(att, def)`。
- `combat/Effects.ts` —— **唯一状态变更入口** `applyEffect(source, target, effect, hooks)`：伤害/治疗/上Buff/驱散全走它。
- `combat/BuffSystem.ts` —— Buff 叠层/到期/周期/属性聚合/行为门（纯函数）。
- `BattleEntry.ts` + `ui/BattleStageView.ts` —— 驱动 + 渲染（色块）。

## 统一属性表 stats
- `CombatStats` 接口列出所有战斗属性：hp/atk/def/range/attackSpeed/critRate/critDmg/dodgeRate/blockRate/blockRatio/dmgBonus/dmgReduce。
- 角色（`stats` 表，按职业）和怪（`enemyTypes[].stats`）共用同一套。
- 单位（Soldier/Enemy）持 `stats` **引用** → 改配置实时生效。

## 伤害公式（减法）
结算顺序：**闪避 → 减法基础(max(atk-def, atk×minDamageRate)) → 暴击(×(1+critDmg)) → 格挡(×(1-blockRatio)) → 伤害加成(×(1+dmgBonus)) → 伤害减免(×(1-dmgReduce))**。
- 暴伤用「加成式」critDmg（可叠加），不用倍率——为装备铺路。

## 如何加一个新战斗维度（标准流程）
1. `CombatStats` 接口 + `BattleConfig.stats`/`enemyTypes` 各行补该字段。
2. `CombatFormula.calcDamage` 里加结算逻辑。
3. 调参面板 `ConfigPanel` 的 `STAT_META` 补一行 → 自动出滑块。
（攻速类「行为」字段如 fireInterval/moveSpeed 在 `classes` 里，不在 stats。）

## 如何加一个新 Buff（标准流程）
1. `buff.xlsx` 加一行（时长/叠层/周期效果/属性修正/flags/驱散标签）→ `npm run config`。
2. 想让技能挂它：`skill.xlsx` 的 effects 列写 `applyBuff:<id>`（导表期校验引用存在）。
3. 属性修正只支持 CombatStats 键且**禁改 hp**；层数线性叠加，聚合经 normalizeStats 钳制。

## 如何加一个被动技能（标准流程）
1. `skill.xlsx` 加一行：`kind=passive`，填 `passiveTrigger`（always/onHit/onHurt/onKill/onCast）、`chance`、`targetMode`（trigger/self/team）、`effects`；主动列必须留空。注意**每职业主/被动合计 ≤2 槽**。
2. 常驻/光环用 `always + applyBuff:<永久buff>`（buff.xlsx 里 `duration=-1`）。
3. `npm run config` → 跑 `test:passive` + `sim:pacing`（被动改变强度，门槛要复跑）。
> 语义红线：普攻命中=近战直击+普攻弹道（技能不算）；受击=直击未闪避（DoT 不算）；DoT 击杀不触发 onKill；被动 proc 出的效果**不再触发被动**（一层防递归）。

## 如何加一种新 Effect（改管线的唯一场景）
1. `config/EffectTypes.ts`：union 加种类 + `parseEffectList` 加编码分支。
2. `combat/Effects.ts`：`applyEffect` 加 case。
3. 补 `tools/effect-test.ts` 用例。

## 技能投递方式（delivery）与控制语义
- `skill.xlsx` 的 `delivery` 列：空=instant（即时结算）；`line:速度[:穿透]` / `arc:速度:重力` / `zone:半径:时长:周期`——弹道命中/场地周期对目标执行该技能的效果列表。投递技能的 `skillCast` 事件即时发出且 `hits` 为空。
- 护栏：`MAX_PROJECTILES=64`、`MAX_ZONES=8`，达上限静默跳过生成。
- 控制语义：**嘲讽**=光环式，敌人贴身攻击优先打最近的 `gate.taunting` 士兵；**沉默**=只禁释放，进度照走、就绪保留待放；**击退**=敌人 +x 钳 halfW、我方 −x 钳左界，远程/治疗钉站位等于免疫。

## 关键不变量（别破坏）
- 近战前压硬保险：坦克位置 ≤ `防线 + contactGap − range`，保证怪永远在坦克前方。
- 远程必须有限射程（否则怪被秒在屏幕外，坦克/治疗失效）。
- 防线 `defenseLineX` 用单位**原始站位 homeX**，不随近战冲出去漂移。
- **改单位状态必须走 `applyEffect`**（伤害/治疗/上Buff/驱散），禁止在别处直接改 hp/buffs；周期 DoT 是唯一例外（`_onBuffPeriodic`，按 srcAtk 快照）。
- 无 buff 单位 `stats === baseStats` 必须保持**同一引用**（ConfigPanel 实时调参靠它）；重算只经 `recomputeDerived`。
- **行军是表现假象**：布阵/防线/刷怪永远在同一套局部坐标，禁止引入世界坐标/镜头系统；`marching` 只是按 `Wave.distance ÷ 最慢 stats.moveSpeed` 的倒计时 + 渲染层背景卷动。
- 移速在 `stats.moveSpeed`（统一属性表），不要往 classes/enemyTypes 加顶层速度字段。
