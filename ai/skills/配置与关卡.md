# 技能 · 配置与关卡

## ⚙️ 数值现在由 Excel 管理（导表管线）

**改数值流程**：编辑 `tools/config-xlsx/battle.xlsx` → 跑 `npm run config` → 自动生成 `assets/scripts/config/battle.config.generated.ts`。

- `assets/scripts/config/BattleConfig.ts` 现在**只保留类型定义**，数值对象 `BattleConfig` 直接引用生成产物。**别手改 BattleConfig.ts 里的数值**（已无数值），也别手改 `.generated.ts`（每次导表覆盖）。
- 生成器 `tools/excel-to-config.ts`：读 6 个 sheet → 产物结构与历史手写版完全一致（BattleManager 引用、ConfigPanel setter 不受影响）。
- 产物 `.generated.ts` **入库**（Cocos 编辑器/构建不会自动跑 npm，不提交别人拉下来缺配置）。
- 种子脚本 `tools/seed-battle-xlsx.ts`：从零重建 xlsx 才跑（极少）；`tools/_e2e-test.ts`：改表→导出→产物更新的回环自测。
- **导表带校验**（缺字段/引用不存在/键不一致/levelIndex 跳号 → 报错阻断；概率越界/count<1 → 警告）。改了 Excel 跑一次 `npm run config`，红字就是哪格填错。

**sheet → 配置块对应**：Stats→stats、EnemyTypes→enemyTypes、Classes→classes、Levels→levels、Misc→散标量(startLevel/combat/layout/bullet/formation)、Scene→scene。roster 取 Stats 行顺序。

> 加新职业要**同时**在 Excel(Stats+Classes) 加行，并改 `BattleConfig.ts` 里 `SoldierClass` 联合类型（手写 union，导表不会自动改它）。

## 配置块结构（类型定义在 `BattleConfig.ts`，数值在 Excel）

## stats —— 小队统一属性表
按职业（tank/dps/healer）一行，列为 `CombatStats` 全部属性。

## enemyTypes —— 怪物类型表（图鉴）
每种怪：`name` / `stats`(CombatStats) / `speed` / `radius` / `attackInterval` / `color`(RGB)。
加新怪：在这里加一个 key 即可，关卡里按 key 引用。

## levels —— 关卡表
```
levels[i] = { name, waveGap, waves: [
  { spawns: [ { type, count, interval, hp? }, ... ] },  // 一波可混多种怪
] }
```
- `type` 对应 enemyTypes 的 key；`hp` 不填用该怪类型基础血量（填了做难度递增）。
- `startLevel` 选默认进哪关；打完该关所有波 = 通关。

## classes —— 职业行为（非战斗属性）
attackType / fireInterval / moveSpeed / advanceLimit / healPerSec / size。

## 其它块
roster（阵容顺序）、layout（站位）、bullet、formation.contactGap（怪贴脸距离）、combat.minDamageRate、scene（背景）。

## 调参面板 ConfigPanel
- 游戏内 HTML 浮层（仅网页预览），右上角，拖滑块实时改、可导出 JSON。
- 战斗属性滑块**自动**按 `STAT_META` × 每个单位生成；加新属性只需在 `STAT_META` 补一行。
- 定位：**临时手感调试**（局内改内存值，不落 Excel）。调好的数值要**手动回填到 Excel** 再 `npm run config` 固化——面板不写回 xlsx。
