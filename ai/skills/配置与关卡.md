# 技能 · 配置与关卡

## ⚙️ 数值现在由 Excel 管理（导表管线）

**改数值流程**：
- 战斗/关卡：编辑 `tools/config-xlsx/battle.xlsx` → 跑 `npm run config` → 自动生成 `assets/scripts/config/battle.config.generated.ts`。
- 装备：编辑 `tools/config-xlsx/equip.xlsx` → 跑 `npm run config` → 自动生成 `assets/scripts/config/equip.config.generated.ts`。
- 掉落：编辑 `tools/config-xlsx/drop.xlsx` → 跑 `npm run config` → 自动生成 `assets/scripts/config/drop.config.generated.ts`。

- `assets/scripts/config/BattleConfig.ts` 现在**只保留类型定义**，数值对象 `BattleConfig` 直接引用生成产物。**别手改 BattleConfig.ts 里的数值**（已无数值），也别手改 `.generated.ts`（每次导表覆盖）。
- 生成器 `tools/excel-to-config.ts`：**多源驱动**——顶部 `SOURCES` 清单描述「哪个 xlsx → 哪个 parser → 哪个产物」，`npm run config` 遍历全部源依次导出，每源独立校验/独立产物。目前有 battle（6 sheet → `battle.config.generated.ts`）、equip（3 sheet → `equip.config.generated.ts`）和 drop（3 sheet → `drop.config.generated.ts`）。
- **加新模块 = 一个模块一张表**：① 写 `buildXxxConfig(wb)` 解析函数；② 在 `SOURCES` 加一行 `{name,xlsxRel,outRel,exportVar,build}`；③ 消费端按 `exportVar` import。主流程不用改。
- 产物 `.generated.ts` **入库**（Cocos 编辑器/构建不会自动跑 npm，不提交别人拉下来缺配置）。
- 种子脚本 `tools/seed-battle-xlsx.ts`：从零重建 xlsx 才跑（极少）；`tools/_e2e-test.ts`：改表→导出→产物更新的回环自测。
- **导表带校验**（缺字段/引用不存在/键不一致/levelIndex 跳号 → 报错阻断；概率越界/count<1 → 警告）。改了 Excel 跑一次 `npm run config`，红字就是哪格填错。

**sheet → 配置块对应**：Stats→stats、EnemyTypes→enemyTypes、Classes→classes、Levels→levels、Misc→散标量(startLevel/combat/layout/bullet/formation/roster)、Scene→scene。roster 默认取 Stats 行顺序；若 Misc 里填 `roster`（逗号分隔，如 `dps` 或 `tank,dps,healer`），则按它作为当前出战阵容。

**equip.xlsx sheet → 配置块对应**：Qualities→品质倍率/中文名/roll 范围/附加词条数量，SlotBonuses→每个部位的基础属性加成，Affixes→高品质装备可抽取的附加词条池；新掉落由 `EquipConfig.calcEquipItemStats(slot, quality)` 计算实际 `EquipItem.stats`。

**drop.xlsx sheet → 配置块对应**：DropGroups→掉落组（`group/itemCount/qualityGroup/slotGroup`），QualityWeights→品质权重组，SlotWeights→部位权重组；运行时由 `DropConfig.rollDropItems(level.dropGroup)` 生成装备。关卡表只放 `dropGroup` 指针，不把品质权重散进 `battle.xlsx`。

> 加新职业要**同时**在 Excel(Stats+Classes) 加行，并改 `BattleConfig.ts` 里 `SoldierClass` 联合类型（手写 union，导表不会自动改它）。

## 配置块结构（类型定义在 `BattleConfig.ts`，数值在 Excel）

## stats —— 小队统一属性表
按职业（tank/dps/healer）一行，列为 `CombatStats` 全部属性。

## enemyTypes —— 怪物类型表（图鉴）
每种怪：`name` / `stats`(CombatStats) / `speed` / `radius` / `attackInterval` / `color`(RGB)。
加新怪：在这里加一个 key 即可，关卡里按 key 引用。

## levels —— 关卡表
```
levels[i] = { name, waveGap, dropGroup, waves: [
  { spawns: [ { type, count, interval, hp? }, ... ] },  // 一波可混多种怪
] }
```
- `type` 对应 enemyTypes 的 key；`hp` 不填用该怪类型基础血量（填了做难度递增）。
- `dropGroup` 是胜利奖励掉落组 id，对应 `drop.xlsx` 的 DropGroups.group；同一关多行必须一致。
- `startLevel` 选默认进哪关；打完该关所有波 = 通关。

## classes —— 职业行为（非战斗属性）
attackType / fireInterval / moveSpeed / advanceLimit / healPerSec / size。

## 其它块
roster（当前出战阵容；可在 Misc.roster 覆盖）、layout（站位）、bullet、formation.contactGap（怪贴脸距离）、combat.minDamageRate、scene（背景）。

## 调参面板 ConfigPanel
- 游戏内 HTML 浮层（仅网页预览），右上角，拖滑块实时改、可导出 JSON。
- 战斗属性滑块**自动**按 `STAT_META` × 每个单位生成；加新属性只需在 `STAT_META` 补一行。
- 定位：**临时手感调试**（局内改内存值，不落 Excel）。调好的数值要**手动回填到 Excel** 再 `npm run config` 固化——面板不写回 xlsx。
